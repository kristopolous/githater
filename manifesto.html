IHATE MANIFESTO

Unix is hard, let's start talking.

INTRODUCTION

I started a githater twitter account 2 years ago out of my frustration with my inability to get the hang of the powerful syntax.

git doesn't stand alone here. I can never remember how to

  • modify my firewall rules
  • do anything with gpg
  • do a POST in curl
  • anything more than the most basic sed
  • use getopt or readline
  • use find to say, find new files in the past day
  • dates, anywhere; sql, the shell, php
  • just about anything with mdadm
  • cut, copy, paste; from the command line. I'm sure it's awesome.

I'm not alone here. But what are we going to do about it?

IHATE

Imagine a tool called ihate. Here is an example:

$ ihate git how do i clone things quickly
Only clone what's necessary to get going instead of everything
git clone --depth=1

The syntax is:

  ihate (program) (description of what you want to do)

And then, like magic, you will get a human readable solution to get things done.

THE MAGIC

IHATE.PY makes RESTful calls to a community driven website.

On the website a very generous person has provided the following:

  • name of program
  • 1 and 2-grams along with a weight for the matching of the sentences
  • a twitter-like description of the task
  • an actual command line
  • a disclaimer and a rank as to how "safe" a command is. As in "Warning: This command reformats your hard drive. Risk: 5/5 Highest"

That's it. It's going to be awesome!
